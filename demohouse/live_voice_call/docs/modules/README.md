## 主链路排障建议
1. 先看前端路由与守卫（token/session 与设备检查）。
2. 再看前端实时链路（录音、WS、事件解析、字幕/消息状态）。
3. 再看后端连接处理与事件流（`handler`、`event`、`utils`）。
4. 最后看持久化和管理面逻辑（`admin_store`、`admin_api`）。

- 每个文档统一包含：职责、入口、接口、依赖、日志、常见故障、验证建议。
