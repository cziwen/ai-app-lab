# frontend/src/routes/check-in.tsx

## 模块职责
- 面试前设备检查页：按顺序完成扬声器、麦克风、摄像头、屏幕共享检测。
- 全部通过后才允许进入正式通话页。

## 入口与调用方
- 路由入口：`/check-in?token=...`。
- 通过 `useSessionAuth` 读写权限状态与媒体流引用。

## 对外接口（导出项）
- `CheckInPage`

## 关键流程
1. 读取设备列表（输出/输入/摄像头）。
2. 按 `speaker -> mic -> camera -> screen` 串行检查。
3. 每个步骤失败会写入错误提示并阻断下一步。
4. 点击“进入面试”前做最终复检（设备仍可用、屏幕共享仍有效）。

## 依赖与配置
- 浏览器能力：`navigator.mediaDevices`、`getUserMedia`、`getDisplayMedia`、`AudioContext`。
- 需要 HTTPS 或 localhost 才能稳定使用完整媒体能力。

## 日志与排障
- 重点看页面错误消息：设备读取失败、权限拒绝、共享类型不符。
- 常见根因：浏览器权限未授权、系统级设备占用、屏幕共享未选“整个屏幕”。

## 常见故障与排查步骤
1. 现象：一直停留在“待检测”。
- 检查浏览器是否弹出权限框并被允许。
- 刷新后重新触发设备枚举。

2. 现象：屏幕共享总失败。
- 必须选择 `displaySurface = monitor`（整个屏幕），窗口/标签页会被拒绝。

3. 现象：进入面试按钮置灰。
- 检查四项权限是否都为 `granted`。
- 检查最终复检是否因设备断开而回退。

## 手工验证
- 按顺序完成四项检测后进入面试；断开任一设备后再次进入应提示失败。
